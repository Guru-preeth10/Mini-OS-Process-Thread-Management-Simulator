🚀 FIRST TIME USAGE
════════════════════════════════════════════════════════════════════════════════

1. CREATE A PROCESS
   - Click "Create New Process" button
   - Enter name (e.g., "Background Tasks")
   - Click "Create Process"
   → Process card appears!

2. ADD A THREAD
   - Click "Add Thread" on process card
   - Select "Print Task" or "Calculation Task"
   - Fill in parameters
   - Click "Add Thread"
   → Thread appears on the card!

3. RUN THE PROCESS
   - Click "Run" button
   - Watch the execution window open
   - See live output of threads
   - See completion statistics
   → Process executed successfully!

4. VIEW SCHEDULER
   - Click "Scheduler" tab
   - See all processes and their states
   - View execution timeline
   → Full visibility of your system!

🔌 BACKEND INTEGRATION
════════════════════════════════════════════════════════════════════════════════

WORKS IN TWO MODES:

Mode 1: Standalone (Default)
─────────────────────────────
✓ UI works without any backend
✓ All interactions are simulated
✓ Perfect for testing the UI
✓ No configuration needed

Mode 2: Connected to Java Backend (Optional)
──────────────────────────────────────────────
✓ Real API calls to Java backend
✓ Actual process execution
✓ Live output display
✓ Set API_BASE in js/app.js

CONNECT TO BACKEND:
───────────────────
1. Edit js/app.js (line ~10)
2. Change: const API_BASE = 'http://localhost:8080/api';
3. Match your Java backend port
4. Save and reload UI
5. Done! Now connected!

REQUIRED ENDPOINTS:
────────────────────
POST /api/createProcess
POST /api/addThread
POST /api/runProcess
GET /api/getProcesses
GET /api/getSchedulerInfo

🐛 TROUBLESHOOTING
════════════════════════════════════════════════════════════════════════════════

"Page shows blank"
──────────────────
→ Check browser console (F12)
→ Make sure browser supports ES6+
→ Try different browser
→ Clear cache: Ctrl+Shift+Delete

"Buttons don't respond"
────────────────────────
→ Open DevTools (F12)
→ Check Console for errors
→ Run: debugState() to check app state
→ Refresh page (Ctrl+R)

"Can't connect to backend"
───────────────────────────
→ Check API_BASE URL in js/app.js
→ Verify Java backend is running
→ Check Network tab in DevTools
→ Ensure CORS is enabled on backend
