🎨 Mini OS Control Center - Setup Guide

Welcome! This guide will help you set up and run the professional web UI for the Mini OS simulator.

🚀 Quick Start

Option 1: Simple (No Server Setup)

For testing the UI locally:

  1. Open index.html in your browser
  2. The UI will load in demo/offline mode
  3. All features work except backend API calls

Option 2: With Local Server (Recommended)

Using Python's built-in HTTP server:

# From the ui directory python -m http.server 8000 # Then open browser: # http://localhost:8000

Option 3: Connect to Java Backend

Full integration with the Mini OS Java application:

# Step 1: Start Java backend cd MiniOS java os.Main # Step 2: Update API endpoint in js/app.js const API_BASE = 'http://localhost:8080/api'; # Step 3: Open UI in browser # http://localhost:8000

🎯 Using the UI

Create a Process

  1. Click "Create New Process" button
  2. Enter a process name (e.g., "Background Tasks")
  3. Click "Create Process"
  4. Process card appears in the list

Add a Thread

  1. Click "Add Thread" on any process card
  2. Select task type:
    • Print Task: Prints a message N times
    • Calculation Task: Performs math (+ - * /)
  3. Fill in task parameters
  4. Click "Add Thread"

Run a Process

  1. Click "Run" button on process card
  2. Execution window appears with live output
  3. Watch threads execute concurrently
  4. See execution statistics on completion

🔌 Backend Integration

Current Setup (Mock Mode)

The UI works in standalone mode with simulated data. All interactions stay local.

Connect to Java Backend

To use with the real Mini OS Java application:

⚠️ Important: Your Java backend must provide REST API endpoints.

Required API Endpoints

Endpoint Method Purpose Request Body
/api/createProcess POST Create new process { name: string }
/api/addThread POST Add thread to process { processId, taskType, ... }
/api/runProcess POST Execute process { processId }
/api/getProcesses GET Get all processes None
/api/getSchedulerInfo GET Get scheduler stats None

Configuration

Edit js/app.js to change the API base URL:

// Line ~10 in js/app.js const API_BASE = 'http://localhost:8080/api'; // Change to your backend URL: const API_BASE = 'http://your-server.com:port/api';

🐛 Troubleshooting

UI Doesn't Load?

  • Check browser console for errors (F12)
  • Verify index.html path is correct
  • Try a different browser
  • Clear browser cache

API Calls Fail?

  • Verify Java backend is running
  • Check API_BASE URL in js/app.js
  • Ensure CORS headers are set on backend
  • Check Network tab in DevTools (F12)

Buttons Don't Work?

  • Open browser console for JavaScript errors
  • Check that js/app.js is loaded
  • Try refreshing the page
  • Run debugState() in console

Debug Mode

Open browser console and type:

debugState() // Shows current app state
